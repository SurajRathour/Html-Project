Platform Overview

Analyze and enhance your audio, at scale. To use the dlby.io Media APIS, you'll need to:

  • Get your API Key
  • Configure your cloud storage to be readable and writeable
  • POST to an APi endpoint to start a request on your media
  • Poll and GET an API endpoint to see when the job completes

You'll find additional details for working with your API Keys and making Async Requests below. For additional help with cloud storage review the Media Input/Output section.

Get your API Key

API keys are required to make calls to the Media APIs You can get an API key assigned to you by creating a developer account and accessing it from your dashboard. The API key is a unique identifier that is assigned to you. You'll want to keep this secret for any applications you create.

To use the API key you will include the key in a header called x-apikey as part of any requests. This might look like:

curl --header X-api key YOUR API KEY

You'll need to provide this configuration in whichever dient languages you prefer to use such as Python, JavaScript, or Java.

Async Requests

The Media Service Apis work asynchronously meaning that your first HTTP request initiates a job which is assigned a unique job id. You will need to make a second request to GET the status of that job.

The expected status values includes

  • uccess - this status indicates you can retrieve your media at the output URL you provided on the nitial request
  • Cunning - your media is being processed, check back again soon
  • Fending - your media is waiting for an available resource to run it
  • Failed - there was a problem and you'll see an error with some additional notes about what the cau might be

When a Job completes successfully your media will be available from the output URL specified in the initial request.

The series of call and resulting status could follow this flow:

Step Method Request Response
1 POST Provide input and output Receive job_id
2 GET Provide job_id Receive status Pending
3 GET Provide job_id Receive status Pending
4 GET Provide job_id Receive status Pending
5 GET Provide job_id Receive status Pending
6 GET Provide job_id Receive status Pending

This is a common pattern called polling.Depending on the size of your media files you may need to wait a few moments to get the results of processing. ideally you will see a Success result suggesting the media is ready for your review. In some cases you may get a Failed result if for example we were unable to read or write to the locations you provided as input and output.

To learn more about the input and output parameters please see Media Input/Output.
